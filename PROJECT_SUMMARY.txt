4. KNOWN BUGS & INCOMPLETE MODULES
================================================================================

TRADINGVIEW WIDGETS
  - Fixed: removed script.async to avoid querySelector null errors
  - May still flash/reload on React Strict Mode double-mount in dev
  - Depends on third-party CDN (s3.tradingview.com) — offline = broken widgets
  - Multiple widgets on Dashboard can be slow or rate-limited by TradingView

FINNHUB
  - Free tier rate limits — many symbols / fast polling may fail
  - useFinnhub skips polling when market closed (no live updates after hours)
  - WebSocket max 50 symbols; symbol format may need exchange prefix for some tickers
  - BRK.B and similar symbols may need different Finnhub symbol format

SUPABASE / AUTH
  - Admin actions on auth.users require RPC (phase11-admin-rpc.sql) — must be run
  - User must re-login after role/metadata changes
  - Google OAuth requires manual setup in Supabase dashboard
  - Delete user via admin is irreversible

TRADING LOGIC
  - Paper trading only; no order slippage/fees
  - is_frozen checked in BuySellModal only (not all code paths)
  - History realized P&L uses simplified avg buy (not FIFO lot matching)
  - undoLast reverses last DB transaction; edge cases if portfolio manually edited

LEADERBOARD
  - Requires phase10-leaderboard.sql RPC
  - Portfolio value uses live Finnhub prices when available

UI / UX
  - Landing page is minimal (title + CTA only)
  - Large JS bundle (~960KB) — no code splitting
  - No global React Error Boundary
