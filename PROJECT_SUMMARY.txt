0. PROJECT OVERVIEW
================================================================================

TradeX is a browser-only paper trading platform. Users get a virtual $10,000
wallet on signup, trade 100 predefined US stocks at live (or cached) Finnhub
prices, and compete on a leaderboard. Admins manage users, fund requests, and
audit actions via Supabase RPCs protected by JWT role metadata.

Architecture (high level):

  Browser (React SPA)
    ├── Supabase Auth + Postgres (RLS) — users, wallets, trades, watchlist
    ├── Finnhub REST — quotes, news, candles (client-side API key)
    ├── Finnhub WebSocket — live prices on Watchlist (max 50 symbols)
    ├── Finnhub CDN — company logos (static2.finnhub.io stock_logo URLs)
    └── TradingView CDN widgets — charts, tape, news, market overview

Documentation (single sources of truth):
  README.md                    — Quick start, deploy, links
  supabase/SUPABASE_SETUP.md   — ALL Supabase steps (SQL, Auth, emails, admin)
  PROJECT_SUMMARY.txt          — This file (deep technical reference)

Source scale:
  ~90+ project files (excluding node_modules / dist)
  18 user-facing routes + admin + public auth + 404

4. ROUTING
================================================================================

PUBLIC
  /                  Landing
  /login             Sign in / sign up
  /verify-email      Post-signup confirmation + resend
  /forgot-password   Request reset link
  /reset-password    Set new password from email
  *                  NotFound

USER (ProtectedRoute → Layout → Navbar + TickerTape)
  /dashboard         Stats, TV widgets, recent transactions, portfolio pie
  /search            Stock search + company logos on cards
  /portfolio         Holdings, P&L chart, buy/sell
  /watchlist         WebSocket live prices + chart
  /history           Filters, CSV, undo last trade
  /wallet            Balance, fund requests
  /leaderboard       get_leaderboard() RPC + live marks
  /profile           Account overview, stats, quick links
  /settings          Display name, password, preferences, logout
  /stock/:symbol     Chart, symbol info, news, trade, watchlist

ADMIN (AdminRoute — user_metadata.role === 'admin')
  /admin, /admin/users, /admin/wallet, /admin/analytics, /admin/logs

Guards:
  ProtectedRoute → /login if no session; BannedScreen if is_banned
  AdminRoute → admin only
  is_frozen → blocks trading in BuySellModal (not global route block)
